## 객체
이름과 값을 하나의 쌍으로 묶은 데이터 타입이다. 하나의 쌍을 **프로퍼티**라고 하며 이름을 key라고 합니다. 생성하는 방법은 두가지가 있는데 객체 리터럴을 이용하여 생성하는 방법을 정리하겠습니다.
<!--more-->

<br>

* 객체 리터럴 로 객체 생성하기

```js
let card = { suit: "하트", rank = "A" };

let card = { 'suit' : '하트', "rank" = "A"};

let card = {};
```

로 대입이 가능하며 이렇게 대입된 값의 표현식은

```js
card.suit // 하트
card["rank"] // A
```

값의 추가의 경우

`card.value = 14;`로 추가 할 수 있다.

<br>

* in 연산자

`"suit" in card`를 통해 프로퍼티의 객체안의 존재 여부를 확인 할 수 있다. 주의 해야 될건 객체가 object 객체를 상속받은 속성이기에 상위 객체의 프로퍼티가 존재하게 된다. 

이처럼 객체가 객체를 상속받는 것 또한 가능하다.

```js
let circle = {
	center: { x: 1.0; y: 2.0},
	radius = 2.5
};
```

<br>

* 객체의 참조 개념

객체 타입의 값을 변수에 할당 하면 메모리 주소가 저장되기에 
`let a = card;`를 하게 되면 `a` 로  card객체의 프로퍼티 주소를 읽거나 수정가능해 진다.

다른 언어에서 처럼 heap영역에 list가 할당되는 것과 비슷한 개념인것 같다. 하지만 js는 브라우저 내에서 돌아가는 언어이기 때문에 메모리의 할당부분에서 차이가 있는것 같다.

<br>

## 함수
수학과는 다르게 함수란 어떠한 작업을 실행하는 하나의 수행이라고 봐야한다. 하나의 함수에서는 하나의 작업만을 하게 끔 만드는 것이 중요하다.

cf) 줄바꿈에 유의 해서 사용해야 한다. js는 ; 이 없이도 하나의 줄을 끝맻음 으로 인식하기에 return 뒤에 줄바꿈을 하지 않기로 하자.


* 함수의 hoisting 

변수와 마찬가지로 js 에서는 작성된 순서에 상관없이 실행이 가능하다. 

* js에서의 함수의 특징

js에서는 함수가 하나의 객체로써 함수 명을 이름으로 함수를 객체로써 가지는 하나의 프로퍼티이다. 따라서 위에서와 같이 변수값에 함수를 할당하는 것도 가능하다.

```js
function square(x) { return: x * x;}

sq = square;
console.log(sq(5)) // 25
```

가 가능해 진다. 이러한 특성으로 고차 함수를 구현 할 수 있게 되는데 뒤에서 정리 하겠습니다.

### 참조(메모리의 주소)의 주의 점

처음에 이해하기 어려웠던 내용인데 js 에서도 똑같은 개념이 나왔네

함수는 원시값을 받아 올때와 객체를 인수로 넘겼을 때의 결과 값이 다르다. 위에서 정리한 참조 개념을 토대로 객체란 주소값을 그대로 가지고 있기 때문에 함수가 진행 후에는 아예 주소를 가지고 함수 로직이 실행되기에 값 자체가 달라지게 된다. 

```js
function add1(x) {return x = x + 1};

let a = 3;
let b = add1(a);

console.log(`a = ${a}, b = ${b}`); // a = 3, b = 4
```

```js
function add1(p) {p.x = p.x + 1; p.y = p.y + 1; return p;}

let a = {x: 3, y: 4};
let b = add1(a);

console.log(a,b); 

// Object {x: 4, y: 5}
// Object {x: 4, y: 5}
```

처럼 함수의 실행 전의 객체 값 자체가 바뀌어 버린다.

객체값이 전달 될 때의 함수의 흐름은 객체값의 복사본이 주소값과 함께 전달 되므로(객체 자체가 주소값을 가지고 있는 메모리 영역이기에) 수정된 값이 주소값에의 하여 기존의 객체에 할당된다.

<br>

* 인수 여러개를 우아하게(?) 전달하는 방법

책에서 소개한 이름은 좀 그렇지만 되게 괜찮은 습관 같다. 함수의 파라미터를 전달할 때 굉장히 애를 먹었던 내용이 

-인수의 순서를 착각하기 쉽고, 매번 확인해주는 것이 상당히 번거롭다.

-함수가 받는 파라미터의 개수가 달라지면 로직자체가 달라진다.

인데 객체를 이용하면 굉장히 단순해 진다.

```js
function setBallPro(x, y, vx, vy, radius) {...}
...
setBallPro(0, 0, 10, 15, 5)
```
 
처럼 개수와 순서를 맞춰야 하는 것이 객체를 활용하면

```js
let paramemters = {
	x: 0,
	y: 0,
	vx: 10,
	vy: 15,
	radius: 5
};

function setBallPro(para) {...}
...
setBallPro(parameters);
```

를 하게 되면 객체의 주소값 때문에 객체값들 자체가 변하는 특징이 있지만 훨씬 가시적이다. 신경써야할 부분들도 줄어들고

## 함수 리터럴로 함수의 정의 

`let square = function(x) {return x * x;};`

와 같이 함수의 선언이 이루어 질 수도 있으며 

* 뒤에 반드시 ;이 붙어야한다.

* 함수의 hoisting이 유효하지 않다. 

인데 나는 아직까지 hoisting의 개념이 왜 쓰이는지 이해가 가지는 않는다. 그냥 로직대로  선언부가 위에 있는 형식으로 코드를 짜는것이 좀더 보기 좋아 보인다.

## 변수의 지역성

변수의 지역성(Scope)는 여타의 언어들과 비슷한 특성을 가지고 있습니다. 지역 변수와 전역 변수가 다릅니다.

* 지역변수와 전역변수가 함수내에서 충돌 할 경우

함수 내부의 지역 변수와 전역변수가 같을 경우 우선적으로 지역 변수를 사용하게 됩니다. 

<br>

* 변수의 hoisting

함수 지역에서도 hoisting의 개념이 유효 함으로 유의해야함
변수의 선언 없이 함수 내부에서 사용 `a = 2;`하게 되면 전역 변수로 선언되기에 유의 해서 사용하고 가급적 let을 통해 선언을 이용하여 변수를 선언해주는 것이 좋을 것 같다.


## let 과 const by ECMAScript 6

* let의 var과의 차이점은 let이 블록 범위를 갖는 다는 것이다. 변수의 hoisting 또한 유효하지 않으며 굳이 변수의 hoisting을 활용할 필요가 없을 것 같아 let을 많이 활용했다. 

<br>

* const 의 경우 상수의 선언으로 변수들과는 달리 값의 변환이 이루어지지 않는다. 초깃값의 설정이 필요하다.




## 객체의 메서드

메서드는 뭐 다양한 언어들에서 조금 씩 다르게 사용되는 것 같지만 
 js 에서는 객체 내부의 property 중에서 함수 객체를 담고 있는 값을 메서드라 칭한다. 

<br>

* 객체 지향 프로그래밍

여기서 정리 하기엔 조금 넓은 개념인 것 같지만 낮은 단계에서 보면 위의 메서드 즉 함수가 객체 내부의 값의 변환, 로직이 해당 객체 내부에서 일어난다는 점에서 객체 지향의 개념이라고 할 수 있을 것 같다.

즉 외부 호출에 의하여 내부 객체의 상태를 바꾸는 개념이다. 

```js
let circle = {
	center: {x: 2, y: 3},
	radius: 2.5,
	area: function () {
		return Math.PI * this.radius * this.radius;
	}
};		//로직이 내부에서 돌아간다.
```

## 객체의 생성자

다른 언어들과는 차이가 있는 개념인것 같다. js에는 class 가 없기 때문에 property의 이름이 선언되지 않은 함수의 작성이 생성자를 대신하게 된다. 밑의 예제 처럼 이후에 new 연산자를 사용하여 객체를 생성할 것이라 기대하여 만들어진 함수를 생성자라고 하는 것이다. 

주조틀의 역할을 하며 초깃값의 설정을 담아 파라미터를 받아와서 각각의 인스턴스를 만드는데 많이 사용한다. 

이러한 생성자는 보통 파스칼 표기법으로 표현한다.

```js
function Card(suit, rank) {
	this.suit = suit;
	this.rank = rank;
} //여기서의 this는 앞으로 생성될 인스턴스를 의미한다.

let card1 = new Card("하트", "A");
let card2 = new Card("클럽", "K");
```

```js
function Circle(center, radius) {
    this.center = center;
    this.radius = radius;
    this.area = function () {
        return Math.PI * this.radius * this.radius;
//메서드 함수안에서 this는 인스턴스의 프로퍼티
    };
}

let p = {x: 0, y: 0};
let c = new Circle(p, 2.0);

console.log(`넓이 = ${c.area()}`);

//넓이 = 12.566370614359172

```

## 배열의 기초

배열은 값의 목록이로 값마다 번호(index) 가 매겨져 있다.  또한 배열은 객체 타입 으로 전에 정리 했던 객체의 특성과 같이 참조의 특성을 띄게 된다. 

`let event = [2, 4, 6, 8];`

index | 값
---|:---
0 | 2
1 | 4
2 | 6
3 | 8

변수의 요소에는 변수와 마찬 가지로 모든 타입의 값이 올 수 있다.

`let various = [3.14, "pi", true, {x: 1, y: 2}, [1, 2]];`



<br>

* length 프로퍼티

전에 js의 활용 만을 했을 때와 다른 언어에서 length와 같은 기능을 사용할때 이게 정확이 무슨 기능이며 이름이 뭔지 헷갈렸는데 객체의 개념을 좀더 깊게 이해 하니 length라는 property라는 것이 배열이라는 내장객체의 하나의 property의 개념임이 자리잡혀서 훨씬 사용함에 있어서 거리낌이 없어졌다.


-length의 길이를 요소의 개수보다 적게 설정하면 이후의 인덱스를 가진 요소들은 짤리게 된다.

<br>

* 생성자를 이용한 배열의 생성

`let a = new Array(2, 4, 6, 8);`


cf) Array 생성자의 인수가 한개이고 양의 정수이면 array의 length를 설정하게 된다.

```js
let x = new Array(3);
console.log(x.length); //3
```

<br>

* 배열의 요소의 참조

배열의 요소하나는 하나의 변수 처럼 사용할 수 있다. 그렇기에 모든 타입의 데이터가 배열에 들어갈 수 있는것이다. 


<br>

* js에서의 array의 특성

다른 언어에서의 array와는 다르게 js의 array는 연속된 메모리애 쌓이는 것이 아닌 객체를 이용하여 다른언어의 array를 흉내낸 것이다. 

따라서 객체의 각 프로퍼티에 인덱스를 문자열로 변환해서 사용한것으로 이해해야 한다.  그렇기 때문에 배열 요소의 사용이 객체를 사용할때와 같이 []를 이용하여 사용한다고 생각하면 된다.

```js
let a = ["A", "B", "C"];

console.log(a["2"]); //C
console.log(a[2]); //C
```


<br>

* 배열의 추가와 삭제

-`a.push()`나 `a[3] = `와 같이 값의 대입을 통해 할 수도 있으며


-`delete a[1]’`연산자를 이용하여 삭제 할 수도 있다.
