이 전의 과정 까지는 나의 로컬 저장소에서의 버전 관리가 이뤄짐의 과정이었고 이렇게 형성된 버전의 관리가 클라우드 상에서 이루어지게 하기 위해서는 Github 클라우드 저장소에의 푸쉬가 필요하다.
<br>
현재 작업 중인 폴더에서 저장할 github의 주소를 알려 주어야한다.
```bash
$git remote add origin 깃헙 저장주소
```
<!--more-->
각 명령어를 보면
<br>

`remote add`

원격 저장소의 추가를 의미한다. 뒤의 origin은 현재 내가 등록하는 원격저장소 주소의 이름이다. origin 대신 아무거나 들어가게 되어도 문제 없다. 
<br>

 이렇게 등록한 원격 저장소에 만든 커밋을 추가 해야 한다.
```bash
$git push origin master
```

`origin`이라는 이름의 원격 저장소를 등록해 놨기 때문에 직접적인 주소 등록 필요없이 `push` 가 가능해 진다. 여기 까지의 과정을 거치게 되면 github 저장소에 파일이 올라간 것을 확인 할 수 있다.
<br>

cf) mac의 경우 관리자 권한 에러가 날 수도 있는데 +master 를 대신 입력해 주면 해결 된다.
<br>
`master`란 브랜치를 의미 하는데 후에 협업과정에서 git의 사용을 다둘때 정리하겠습니다.
