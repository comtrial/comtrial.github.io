### About Git, Github
버전 관리를 할 여타의 프로젝트는 없지만 git을 공부하는 이유는 cli(Command Line Interface) 에 익숙해 지기 위함도 있었다.<br><br>


학기의 데이터베이스 수업때 mariaDB를 설치하는 과정에서 맥 OS 환경 때문에 진짜 너무 고생을 했어서 관련된 공부를 해보고 싶었다. <br><br>
<!--more-->

Git의 기본적인 목적은 내가 하고 있는 프로젝트의 버전관리를 위함이고 그렇게 관리되는 버전관리를 내 로컬호스트, 개인 pc에서 만이 아닌 클라우드 저장소에 올리며 관리 하기 위한 목적이 github이다. <br><br>


Git을 이용하게 되면 과거의 프로젝트, 내가 방금 업데이트한 내용 등이 모두 버전 관리가 된다는 점이 흥미로웠고 , 작업해볼 정적페이지 형성 프로젝트에 적용하여 좀더 익혀 볼 생각이다.<br><br>





### 기본 cli(Command Line Interface)와 폴더에 git 사용여부 결정


<br><br>
기본적인 내용들은 학습하고 그냥 넘어가면 되지만 처음의 막막함을 기억하고 넘어가고싶어서

<br><br>
#### 기본적인 command

코드 | 의미
---|:---:
`pwd` | 현재 폴더 위치 확인
`ls` | 현재 파일 안의 내용(파일, 폴더)의 확인
`cd` | (change directory) 사용할 파일의 변경

<br><br>




### 폴더에 Git으로 버전관리 하겠다고 선언하기
<br><br>

위의 command를 이용하여 terminal 을 통해서 버전관리를 하고자 하는 폴더에 들어간다. 그후 그 폴더 안에서
<br><br>

```bash
$ git init
```
<br><br>

 로컬 저장소의 프로젝트 폴더에 empty git 저장소를 만들어줌을 의미한다.<br><br>



여기까지의 과정이 프로젝트를 버전관리할 빈 Git 저장소를 형성해주는 과정이다.
