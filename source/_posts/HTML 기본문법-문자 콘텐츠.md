### `<ol>,<ul>, <li>`
-ol, ul 은 자식으로 li 만 포함 가능

-li 는 단독으로 사용할 수 없으며, `<ol>` ,`<ul>` 의 자식으로 포함되어야 

-ol의 항목의 순서는 중요도를 의미 할 수도 있다.

-ol tag 안에 ol, ul tag 를 또 넣을 수도 있다.

-ol tag 에서 type을 설정 할 수 있다. 시작 숫자를 지정 할때도 type을 이용하면 된다.
<!--more-->
ex)
`<ol type=“i”>`

-ol tag에 reversed 라는 속성을 입력 할 수도 있다. 순서는 그대로 지만 목록들의 중요 순서도를 반대 시킬 수도 있다.
`<ol type=“i” reversed`
cf) reversed = “reversed” 와 같이 속성의 이름과 값의 이름이 같을 경우 그냥 속성의 이름만을 입력해주는 것 으로 속성값을 입력 할 수 있다.

-li 에서도 value 라는 속성을 이용하여 시작 숫자를 바꿔 줄 수도 있다.
<br>


### `dl, dt, dd`
: 용어의 집합 
-dl 안에는 dt,dd만이 들어가야 한다.
 
-dl 안에는 div 태그가 들어갈 수 없기 때문에 스타일링 하기가 수월하지 않다. 
따라서 밑에 내용과 같이 li 와 dfn 태그를 이용하여 스타일링을 할수 있다.

```
<dt>Coffee</dt>
<dd>coffee ~~~~</dd>

=>

<li>
    <dfn>Coffe</dtn>
    <p>coffee ~~~
</li>
```


<br>

* `<p>`
-정보통신 보조기기에 내장되어있는 다음 문단으로 넘어가는 기능들을 이용할 수 있게 하여 접근성을 높혀준다.


* `<hr/>`
-수평선이 생기게 되는 데 이것을 시각적인 효과만을 위해 사용해서는 안되고, 의미적인 관점에서 사용해야한다.

-문단의 분리(주제)를 위해 설

-css 를 통해 수평선을 없애 줄 수도 있다.
```
hr {
	border:none;
}
```

-block level 이기 때문에 hr 에 스타일 2px dashed red 를 넣게 된다면 상하좌우 모두에  박스형태로 존재 하게 된다.
따라서
```
hr {
	border:none;
	border-top: 2px dashed red;
}
```
이런 작업을 통해 해줘야 한다.

* `<br/>`
-코드 상에서 줄 바꿈을 사용한다고 해서 웹상에 줄바꿈이 되어 나오는 것은 아니기에 위의 태그를 이용하여 줄바꿈을 해 준다.


* `<pre>`
-br 태그때 보았듯 코드상에서의 서식 형태(띄어쓰기, 줄바꿈)가 웹상에 바로 표시 되지 않기에 그런 서식 형태를 그대로 유지 시켜주는 태그 이다.

-기본적으로 monospace 글꼴 계열로 표시됨
(monospace란 글자 각각의 가로너비가 같은 것을 말한다.)

-tab 마저도 인식 되기에 원래의 코드 형식 맣고, 쓰고자 하는 문단 바로 앞에 `<pre>` 를 붙여줘야 한다.
`<pre>동해물과 백두산이 마르고 …`



* `<brockquote>`
-인용할 내용을 감싸는 태그이다.

-cite 라는 속성을 이용하여 인용해온 주소를 입력해 줄 수도 있다.
