HTML의 기본 원리

우선 큰 틀에서의 html을 이해해 보려 한다. html 은 문서의 형식을 구축해주는 과정이다. 문서에 여기는 머릿글이다. 여기는 이미진데 이제 뭐를 겻들인..식의 요소의 설명이나, 영역의 설명이 포함되게 된다.

주의해야 할 점이 html 에 스타일을 입히기 위해서 div로 막 여기저기 사용하는 경우를 조심해야 한다. html 을 선택대상으로의 구분 보다는 의미를 가진 요소들로 구축해야 한다.

그렇기에 각 요소의 암기보다는 html을 작성 할 시에 흐름에 조금더 집중하여 정리하도록하겠다.

문서의 범위

— HTML

: HTML 문서의 전체 범위를 의미 한다.웹 브라우저가 해석해야 할 HTML문서가 어디에서 어디까지 인지를 알려주는 역활을 한다.

— HEAD( 정보 의 범위)

: 웹 브라우저가 해석해야 할 HTML 문서의 정보 범위를 지정한다. 여기서의 정보에는 웹 페이지의 문자 인코딩 방식, 작성자, title, 연결할 외부 파일의 위치, 기본 세팅 값 들을 말한다.’화면을 구성하기 위한 기본 설정’ 이라고 표현 할 수 있다.

— BODY(구조 범위)

: 구조는 사용자가 화면을 통해서 볼 수 있는 내용(컨텐츠)의 형채나 레이아웃 등을 의미하며 로고, 헤더, 푸터, 내비게이션, 메뉴, 버튼, 입력창, 팝업, 광고 등 보이는 모든 것들이 구조에 해당합니다.
구조는 BODY 범위 안에서만 생성합니다.

— DOCTYPE(DTD, 버전의 지정)

: DOCTYPE(Document Type Definition)읜 마크업 언어에서 문서 형식을 정의한다. 이는 웹 브라우저에 우리가 제공할 HTML문서를 어떤 HTML버전의 해석 방식으로 구조화 하면 되는지를 알려줍니다.(HTML은 크게 1, 2, 3, 4, X-, 5 버전이 있습니다)
