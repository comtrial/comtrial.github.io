JS 문법- 언어적 특성

js의 언어적 특성

  • js는 컴파일 언어와는 다르게 인터프리터를 이용하여 컴파일된 기계어를 한번에 읽어내려가는 것이 아닌 문장단위로 번역하며 실행시키는 언어이다. 그렇기 때문에 처리속도가 느리다는 단점이 있지만 browser의 의 성능 향상(크롬의 8) 덕분에 실행 속도가 매우 빨라지게 되었다.


  • 여타의 언어들과는 다르게 실행전에 변수의 타입을 결정하지 않고, 실행하는 도중에 변수에 저장되는 변수 타입이 동적으로 바뀐다. 따라서 c#과 java 등과 다르게 여러 변수 타입(int32…)등이 필요하지 않다.


  • 함수가 일급 객체이다.

js의 함수는 객체이기 때문에 함수에 함수를 파라미터로 넘길 수 있다. 이 특성을 이용하여 고차함수의 구현이 가능해 진다.

  • 토큰

토큰이란 구문파싱을 할 때 의미를 가지는 구문의 분리를 의미하는데 여타의 책에서는 설명해 주진 않았던 기초적인 부분에 대한 설명까지 있어서 좋았다.

컴퓨터가 이해 할 수 있는 단위들이 있지만 코드는 컴퓨터 뿐만 아니라 사람도 읽기 때문에 코드의 가독성은 매우 중요시 여겨지는 부분이다. 이에 대한 내용은 후에 스타일 가이드를 학습할때 정리 하도록 하겠습니당.

고찰

단지 교수님들의 강의나 프로그래밍 강의 등에서의 단지 활용을 위한 변수, for문의 활용등을 나열한 것 이 아니라 나오게 된 배경과 이 언어에서는 이렇게 쓰이는 것이 더 효율적이기 ‘때문’을 설명해주는 것 같아서 학습내내 즐겁게 공부했던것 같다.
