참조(메모리의 주소)의 주의 점

처음에 이해하기 어려웠던 내용인데 js 에서도 똑같은 개념이 나왔네

함수는 원시값을 받아 올때와 객체를 인수로 넘겼을 때의 결과 값이 다르다. 위에서 정리한 참조 개념을 토대로 객체란 주소값을 그대로 가지고 있기 때문에 함수가 진행 후에는 아예 주소를 가지고 함수 로직이 실행되기에 값 자체가 달라지게 된다.

처럼 함수의 실행 전의 객체 값 자체가 바뀌어 버린다.

객체값이 전달 될 때의 함수의 흐름은 객체값의 복사본이 주소값과 함께 전달 되므로(객체 자체가 주소값을 가지고 있는 메모리 영역이기에) 수정된 값이 주소값에의 하여 기존의 객체에 할당된다.

변수의 지역성

변수의 지역성(Scope)는 여타의 언어들과 비슷한 특성을 가지고 있습니다. 지역 변수와 전역 변수가 다릅니다.

  • 지역변수와 전역변수가 함수내에서 충돌 할 경우

함수 내부의 지역 변수와 전역변수가 같을 경우 우선적으로 지역 변수를 사용하게 됩니다.


  • 변수의 hoisting

함수 지역에서도 hoisting의 개념이 유효 함으로 유의해야함
변수의 선언 없이 함수 내부에서 사용 a = 2;하게 되면 전역 변수로 선언되기에 유의 해서 사용하고 가급적 let을 통해 선언을 이용하여 변수를 선언해주는 것이 좋을 것 같다.

let 과 const by ECMAScript 6

  • let의 var과의 차이점은 let이 블록 범위를 갖는 다는 것이다. 변수의 hoisting 또한 유효하지 않으며 굳이 변수의 hoisting을 활용할 필요가 없을 것 같아 let을 많이 활용했다.

  • const 의 경우 상수의 선언으로 변수들과는 달리 값의 변환이 이루어지지 않는다. 초깃값의 설정이 필요하다.

js 에서는 객체 내부의 property 중에서 함수 객체를 담고 있는 값을 메서드라 칭한다.


  • 객체 지향 프로그래밍

여기서 정리 하기엔 조금 넓은 개념인 것 같지만 낮은 단계에서 보면 위의 메서드 즉 함수가 객체 내부의 값의 변환, 로직이 해당 객체 내부에서 일어난다는 점에서 객체 지향의 개념이라고 할 수 있을 것 같다.

즉 외부 호출에 의하여 내부 객체의 상태를 바꾸는 개념이다.

배열의 기초

배열은 값의 목록이로 값마다 번호(index) 가 매겨져 있다. 또한 배열은 객체 타입 으로 전에 정리 했던 객체의 특성과 같이 참조의 특성을 띄게 된다.

let event = [2, 4, 6, 8];

index 값
0 2
1 4
2 6
3 8

변수의 요소에는 변수와 마찬 가지로 모든 타입의 값이 올 수 있다.

let various = [3.14, "pi", true, {x: 1, y: 2}, [1, 2]];


  • length 프로퍼티

전에 js의 활용 만을 했을 때와 다른 언어에서 length와 같은 기능을 사용할때 이게 정확이 무슨 기능이며 이름이 뭔지 헷갈렸는데 객체의 개념을 좀더 깊게 이해 하니 length라는 property라는 것이 배열이라는 내장객체의 하나의 property의 개념임이 자리잡혀서 훨씬 사용함에 있어서 거리낌이 없어졌다.

-length의 길이를 요소의 개수보다 적게 설정하면 이후의 인덱스를 가진 요소들은 짤리게 된다.


  • 생성자를 이용한 배열의 생성

let a = new Array(2, 4, 6, 8);

cf) Array 생성자의 인수가 한개이고 양의 정수이면 array의 length를 설정하게 된다.


  • 배열의 요소의 참조

배열의 요소하나는 하나의 변수 처럼 사용할 수 있다. 그렇기에 모든 타입의 데이터가 배열에 들어갈 수 있는것이다.


  • js에서의 array의 특성

다른 언어에서의 array와는 다르게 js의 array는 연속된 메모리애 쌓이는 것이 아닌 객체를 이용하여 다른언어의 array를 흉내낸 것이다.
