JS 활용 DOM 객체의 선택

문서의 제어

DOM의 이해

너무나 많이 나오는 이 DOM이라 함은 문서를 제어할 수 있게 해주는 API이다.
렌더링 엔진이 해석한 html문서 구문을 Document 객체가 관리하게 되어 해다 프로퍼티들을 가지고 JavaScript에서 제어 할 수있게 되는 것이다.

각 document 객체의 요소들은 노드 로 표현이 되며, 각각의 이 노드들은 html에서 사용되는 태그요소들의 특징을 담고 있다.(ex) 후손, 형제, 클래스, id …)

node객체 가져오기

웹에서 구동 되는 js는 결국에는 document 문서 즉, html을 제어하는 것이기에 html 관련 요소 태그, 클래스, 아이디 등의 요소들로 컨트롤하기 위해서는 가져오는 방법들을 알아야한다. css에서는 선택자를 사용하는 방법을 사용하는 것 처럼 js에서의 선택하는 방법을 정리해 보겠습니다.

html을 가볍게 여기지 않고 좀 노력을 기울여서 공부한것이 많이 도움이 되었던 것 같다. 어떤 요소들이 어떤 흐름으로 작성되어 내려가는지가 조금은 눈에 잡혀서 선택자의 흐름이 어떻게 가애 할지가 조금은 잡혔다.

css에서 선택자를 이용하는 방법 또한 비슷하기에 흐름이 같이 가져 가되 차이점에 조금 주의 하여 학습하였다. 자동완성의 기능으로 솔직히 각 코드의 암기 보다는 요소 선택자를 어떻게 선택하고 css에서의 nth-child개념등의 선택하는 방법에 대한 흐름이 어떻게 작성되어 사용되는 지를 파악하려고 노력했다.

js에서 밑의 방식으로 가져오게된 요소들은 요소객체 로써 하나의 객체로 할당되게 된다. 이렇게 할당해온 요소객체의 프로퍼티들에서 각 요소의 속성값등을 포함하고 있어 읽기와 수정이 가능하게 된다. 따라서 HTML을 제어하는 역할로서의 js는 이렇듯 제어하고자 하는 요소의 요소객체로의 할당으로 시작되는 것이다.


id속성값으로 node 가져오기

우선 html 문서안에서 id값들은 곂치지 않고 유일 하기 때문에 해당 선택 후에 고려할 것이 없고 간편한 방법이다.

document.getElementById(id값)


Tag, Class로 nodelist 가져오기

해당 선택으로 가져오게 되는 요소들이 유사배열 객체가 되는 방식들로 하위요소의 개념을 사용할 수도 있게 된다. for문이나, 배열의 개념 처럼 꺼내 쓸수 있게 된다.

CSS선택자와 다른 js의 방식으로 문서를 파싱하여 가져오는 개념이기에 document의 내용이 변경되면 nodelist의 요소들도 자동으로 변경되게 된다.

  • Tag 이름 으로의 선택

태그의 이름으로 node를 가져오는 방식이다.
document.getElementsByTagName(tag명)

해당 태그가 여러번 쓰이기에 가져오는 값이 유사배열 객체로써 NodeList 객체가오게 되며 가져온 객체 요소 하나의 선택은

처럼 객체 배열 처럼 뽑아서 사용할 수 있다.


css에서 하위 요소의 선택을 다루는 것 처럼 js에서도 해당 요소안의 하위 요소를 찾아 가는 개념처럼 사용할 수 도 있다.

이 처럼 NodeLIst로써 가져오게 되기 때문에 이러한 컨트롤이 가능해 진다.


  • class 로의 선택

class 속성값은 html에서 쓰이듯이 여러개의 속성값이 공백 으로 구분되어 들어갈 수 있기 때문에 하나하나의 요소값을 선택하여 들여올수 있다.


CSS 선택자로 node 가져오기

위의 방식과는 다르게 css 선택자의 문법을 가져와서 사용할 수 있는 기능이다.
querySelectorAll(선택자)를 사용하면 가르키는 요소들의 nodelist를 반환하지만
위의 방식들 과는 다르게 호출된 시점에서의 일치하는 요소들을 반환한다.
