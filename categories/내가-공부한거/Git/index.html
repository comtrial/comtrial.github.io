Git Gihub에 배포하기

이 전의 과정 까지는 나의 로컬 저장소에서의 버전 관리가 이뤄짐의 과정이었고 이렇게 형성된 버전의 관리가 클라우드 상에서 이루어지게 하기 위해서는 Github 클라우드 저장소에의 푸쉬가 필요하다.

현재 작업 중인 폴더에서 저장할 github의 주소를 알려 주어야한다.

각 명령어를 보면

remote add

원격 저장소의 추가를 의미한다. 뒤의 origin은 현재 내가 등록하는 원격저장소 주소의 이름이다. origin 대신 아무거나 들어가게 되어도 문제 없다.

이렇게 등록한 원격 저장소에 만든 커밋을 추가 해야 한다.

origin이라는 이름의 원격 저장소를 등록해 놨기 때문에 직접적인 주소 등록 필요없이 push 가 가능해 진다. 여기 까지의 과정을 거치게 되면 github 저장소에 파일이 올라간 것을 확인 할 수 있다.

cf) mac의 경우 관리자 권한 에러가 날 수도 있는데 +master 를 대신 입력해 주면 해결 된다.

master란 브랜치를 의미 하는데 후에 협업과정에서 git의 사용을 다둘때 정리하겠습니다.

Git 버전생성

About Git, Github

버전 관리를 할 여타의 프로젝트는 없지만 git을 공부하는 이유는 cli(Command Line Interface) 에 익숙해 지기 위함도 있었다.

학기의 데이터베이스 수업때 mariaDB를 설치하는 과정에서 맥 OS 환경 때문에 진짜 너무 고생을 했어서 관련된 공부를 해보고 싶었다.

Git의 기본적인 목적은 내가 하고 있는 프로젝트의 버전관리를 위함이고 그렇게 관리되는 버전관리를 내 로컬호스트, 개인 pc에서 만이 아닌 클라우드 저장소에 올리며 관리 하기 위한 목적이 github이다.

Git을 이용하게 되면 과거의 프로젝트, 내가 방금 업데이트한 내용 등이 모두 버전 관리가 된다는 점이 흥미로웠고 , 작업해볼 정적페이지 형성 프로젝트에 적용하여 좀더 익혀 볼 생각이다.
