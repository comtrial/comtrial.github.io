generator

이터레이터의 반복처리를 지원하는 함수이다.

함수의 return값에 해당 하는 값들을 반복 처리 했을 경우에 next()에 해당 되는 값들이 나올 수 있는, 반복적으로 실행할 경우 반복적인 값들을 뱉어 낼 수 있는 함수이다.

반환값이 이터레이터이며, 바로 실행되는것이 아니다. return에 해당되는 yield값이 iterator의 value값으로 저장되는 것이다.

반복의 흐름을 제어하는 역할을 하며 반복적인 행위의 단위 마다, 나오는 return값, generator에서는 yield값을 제어하기 위함이다.

함수의 정의

함수의 정의 방법은 4가지가 있는데 그중 hoisting이 가능한 방식은 함수 선언 식으로 정의 하는 방법이다. 나머지는 함수의 할당이 호출 보다 선행되어야 한다.

  • 함수 선언문으로 정의 하는 방법
    function suquare(x) {return x * x;}

  • 중첩함수

js의 언어적 특징중 하나인 중첩함수는 함수의 내부에도 함수가 존재 할 수 있게하며, 외부 함수의 변수를 사용할 수 있는 등, 알고리즈을 좀더 효율적으로 작성할 수 있게 됩니다.


  • 즉시 실행 함수
    뜻 그대로 바로 실행하여 값을 내는 함수 정의로 전역 유효 범위를 해치지 않는 이름 공간의 형성에 쓰인다.

함수의 인수

  • 인수의 생략
    : 인수의 생략을 할 경우에는

논리합 연산자를 사용하면 왼쪽 값이 true일 경우 왼쪽 값을 사용하고, 반대로 false일 경우에 오른쪽 값을 사용하는 연산자를 사용하여 인수 값이 안들어 왔을 경우의 초깃값 설정을 할 수있다.


  • 가변 길이 인수 목록(Arguments 객체)

함수에서 인수를 배열로서 활용할 수 있는 객체로 인수의 갯수가 정해지지 않고 작성된 함수에서 활용 할 수 있다.


this

this를 설명하기 위하여 프로그램의 평가과정을 공부해 봤지만 너무 내용이 깊어 this만을 정리 하고 넘어가려 한다.

this는 함수가 실행되는 시점에 this 값이 결정되며 함수가 호출 될 때 그 함수가 속해 있던 객체의 참조 걊이 된다.

  • cf)
    화살표 함수의 경우 this 값이 함수의 호출에서가 아닌 정의하는 부분에서 결정된다. call, apply를 사용하여도 this 값이 변하지 않는다.
