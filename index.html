WEB

JavaScript 웹 브라우저의 객체

web을 이해하고 공부하는 것이 사실 html, css, js로 그리기에만 그치게 되는 것이 항상 공부를 함에 있어 풀리지 않는 무언가가 있음에 불편했던 것 같다.
그렇기에 web, webpage,web application이 돌아가고, 얹어지게 되는 브라우저의 이해부터 HTTP의 제어, API가 불러지는 원리 등에 관한 조금은 웹의 근간을 공부해 보고자 한다.

브라우저의 이해

Web Browser에서 정적 웹페이지란 문서하나를 표현하는 페이지를 의미한다. javascript가 하는 역할은 이러한 정적인 표현의 문서를 사용자의 입력 대응하여 동작하는 웹페이지 즉, 페이지를 동적으로 해주는 역할이다.

기술적인 측면에서 javascript가 하는 일은 네가지로 분류할 수 있다.

  1. 웹 페이지의 Document 객체의 제어(html, css 스타일 요소)

  2. 웹 페이지의 Window 객체 제어 및 브라우저 제어

  3. 웹 페이지에서 발생하는 이벤트의 처리(click, scroll..)

  4. HTTP를 이용한 통신 제어


이러한 역할을 구현해 주는 웹 브라우저를 하나의 소프트웨어임을 알아야 할 것 같다. 웹이 너무 넓은 개념을 가지고 있는 분야인 것 같아서 방향성을 찾기가 힘들었는데, 웹을 구현해주는 브라우저를 하나의 소프트웨어임을 인지하고 넘어가니 방향성에 대한 이해가 조금은 쉬워 진 것 같다.

이러한 웹을 뿌려주는 브라우저의 기능은

  • 파일 읽기와 쓰기
  • 데이터베이스의 기능
  • 비동기 통신
  • 2, 3차원 컴퓨터 그래픽스
  • 키보드의 입력
  • 마우스의 활용
  • 멀티스레딩

가 있다.

이렇기 때문에 웹 브라우저의 기능이 점점 강력해 지고 일반 데스크톱 애플리케이션을 대체 가능한 수준으로 나아가는 것 같다.


웹 브라우저에서의 자바스크립의 실행순서

웹 브라우저에서 page가 열리면서 어떤 일들이 일어나는 지를 알아봄으로 웹 의 이해를 하고자 합니당.

  1. 웹 페이지가 열리게 되면 먼저 window 객체가 생성됩니다.

  2. 그후에 document 객체가 window 객체의 프로퍼티로써 생성되게 됩니다.
    그리하여 document 객체를 통해 DOM 트리의 구축이 진행되는 것입니다.

  3. HTML 문서를 읽어가며, 작성 순서에 따라 document 객체 요소들과 텍스트 node를 추가 해 갑니다.

  4. HTML에서 script 요소를 만나게 되면 HTML 의 구문 분석을 멈추고 그 즉시에 script요소의 분석을 하게 됩니다.

  5. HTML 구문 분석이 완료되면 document.readyState 프로퍼티 값이 “interactive”로 변합니다.

  6. 웹 브라우저는 document 객체에 DOM트리의 구축이 완료 됨을 알리기 위해서 DOMContentLoaded 이벤트를 발생 시키게 됩니다.

  7. img 등의 요소가 이미지 파일이나 외부 요소에서 읽어와야 한다면 이 시점에 읽어드리게 됩니다.

  8. 모든 리소스들이 읽어지게 되면 document.readyState 프로퍼티 값이 “complete”로 변하게 됩니다. 그리고 마지막으로 웹 브라우저는 window 객체를 상대로 laod 이벤트를 발생 시킵니다.(window.load)

  9. 이후 사용자 이벤트(마우스, 네트워트 이벤트..)를 수신하며, 이벤트가 발생시에 이벤트 처리기가 비동기 적으로 호출되게 됩니다.


이렇게 웹이 열리는 과정을 알아봄 으로 써 그냥 쓰라고 해서 쓰는 줄 알았던 객체를 이해를 기반으로 사용할 수 있게 되는 것 같습니다.

  • window.load 이벤트가 모든 html이 읽어진 후에 발생하는 이벤트임을 알게 되어 html 요소를 조작하는 js의 경우 작동하지 않음을 알고 window.onload 이벤트를 이용할 수 있습니다.

  • load 이벤트의 경우 외부 리소스를 모두 읽어 드린 후에 발생하기 때문에, 사용자가 기다려야 하는 시간이 길어지게 됩니다. 그리하여 그 이전의 단계인 DOMContentLoaded 이벤트에 이벤트 처리기를 등록하여 작성하게 되면 사용자의 경험 시간을 단축 시킬 수 있습니다.

  • async 와 defer : html의 구문 분석에 js 요소가 있을 시에 구문 분석이 중지됨을 앎으로써 구문 분석의 중지를 하지 않게 하는 속성을 알 수 있습니다.

async: script요소가 비동기적으로 실행되게 됩니다. html 분석을 막지 않으며 최대한 빨리 script 요소를 실행 시키게 됩니다.

defer: html 구문의 요소들에 의존하는 script 요소일 경우에 DOM의 구축이 완료 된 후에 일어 나야 하기 때문에 defer와 같이 html 구문을 방해 하지 않으며 DOM 구축 완료 후에 script 요소를 실행 시킬 수 있습니다. 따라서 DOMContentLoaded 이벤트의 대안으로써 사용 될 수도 있습니다.


고찰

공부해본 웹, 브라우저의 작동원리는 우리가 당연시 사용하고 작성하는 것 들과는 다르게 학교에서나 언어의 공부를 할 때 잘 설명을 하지 않거나, 작성되어 있어도 너무 낯설기 때문에 그냥 넘어가는 경우가 많은 것 같습니다.
그렇기에 웹이 열리고 구축되는 과정을 알아봄으로써 여러 api, library가 무엇을 편하게 해주는 지에 대한 이해가 될 것 이라고 생각합니당.

내가 생각하는 개발 공부

개발 공부해보았다.

요번에 복수전공이 되어서 조금은 더 밑 단의 개념을 공부할 수 있게 되어 다행이라고 생각한다. 그럼에도 혼자 공부를 해 온 2달 동안 무언가 부족하다는 생각보다는 내 전공이나 무언가 정해져 있는 과정을 벗어나 내가 공부하고자 하는 공부를 해본것 이 후의 많은 이점을 줄 것 같다는 생각을 했다. 웹의 보여지는 부분을 공부해보고 싶다는 마음이 javascript를 공부하게 했고, java나 python, C#을 배울 때는 와닿지 않았던 객체지향, class화를 필요 하게 했던것 같다.

공부를 위한 공부가 아니라 필요에 의한 공부를 하라고 하지만 사실 어려운 말인것 같다. 학생 신분으로 공부에 있어서 필요를 느끼기란 생각 보다 어려운것 같다. 마냥 놀며 전공대로 가기에도 바쁘고, 그 동안에도 마음은 편하지 않다. 그런 와중에 원하는 방향을 찾게 된것 이 그저 복이지 않을까 싶다.

원하는 바를 가지고 있으면 무언가가 따라온다기 보다는 원하는 바의 길에 무언가는 놓여져있거나, 다가 와주는 것 같다. 이번 방학 학기에도 js를 공부하는 도중 interactive developer 김종민 님이 유튜브를 개설하여 그 프로젝트들을 따라 해보고 추가 해보고, 내가 학습한 내용을 붙여 보고 하는 과정만으로도 생각치도 못한 많은 도움이 되었던것 같다.

공부한 내용을 남기고 싶은 마음의 블로그 개설 과정에서도 git의 사용과 정적 페이지 형성기를 다루었고, 버전의 관리도 경험했다. terminal의 사용도 처음에는 너무 낯설었다.
처음의 공 몇개 튀기는 과정을 아 좀 이런것도 했다고 해야되나 싶은 정도의 project들도 후의 보면 나도 모르게 객체화를 시켜 class를 나누고 export와 import의 과정을 체화시키는 과정이었던것 같다. 뭐라도 만들어 보고 싶은 마음의 page_project는 Slider, Scroll등의 다양한 project를 아무렇지 않게 하고 있었다. 2주전에 공튀기는 거를 만들 때도 쩔쩔 맸는데 말이다.

웃긴것 같다. 처음에는 이런걸 동영상이나 google에서 보고 공부해 보며 나의 project에 적용하는 것 이 그냥 복 붙 해오는 것 과 다를 게 없지 않나 라는 생각이 들어 많이 주저 했지만, 후에 사소한 변경 사항등이나 기능의 추가를 조금은 자연스럽게 하고 있는 모습을 보며 조금은, 아주 조금은 옳은 방향으로 방학을 보냈다는 생각이 들어, 그나마의 웃음이 났던것 같다.

컴공의 전공을 통해 서버와 네트워크, 자료구조등의 밑단의 개념을 매 방학마다 적용해 보려고 해야겠다. 조금 더디지만 옳은 길을 가고 있다고 생각이 들고 싶다.
